# 当前发现

## 2026-04-23

### 已确认

1. `bpmn.js` 基础建模器已在独立项目中跑通。
2. Vite 构建可以通过，说明当前依赖组合至少在打包层面是自洽的。
3. 样式资源路径可被正常解析，包括 BPMN 字体和 minimap 样式。
4. 标准属性面板已成功挂载，右侧独立容器布局可以承载属性编辑区域。
5. `bpmnlint` 已成功挂载，当前采用本地基础规则集，不依赖额外打包插件。
6. `bpmn-js-token-simulation` 已成功挂载，并能与当前 Modeler 一起通过构建。
7. 项目已具备双模式结构：`Modeler` 用于编辑，`Tracking Viewer` 用于验证运行态展示思路。

### 当前问题

1. 当前接入属性面板、Lint、Token Simulation 和 Tracking 原型后，构建产物 `index-*.js` 已增长到约 `1.57 MB`，明显超过默认 chunk 警告阈值。

### 影响判断

1. 当前阶段属于实验项目，这个体积问题不会阻塞继续开发。
2. 当前四层能力已经完成混装，但浏览器交互层是否存在事件冲突仍需实际验证。
3. 未来如果要回流 Odoo，必须做模块拆分、按需加载或功能分层。

### 后续处理建议

1. 下一轮开始可以考虑把属性面板、Lint、Token Simulation 改成动态加载，不必和基础 Viewer/Modeler 强绑定。
2. 优先进入浏览器实测，重点观察模拟模式与属性面板、复制粘贴、右侧面板是否存在交互冲突。
3. 如果浏览器实测稳定，下一阶段应进入“自定义业务属性 Provider”和“真实业务数据映射”。
